## 生产环境 404 错误修复（Vercel 部署）

- **问题**：生产环境（Vercel 部署）中，登录页面点击"获取验证码"按钮时返回 HTTP 404 错误。
- **原因**：前端不知道后端 API 的真实地址，API 请求被发送到了 Vercel（前端部署地），而不是云服务器后端。
- **修复方案**：创建 API 代理路由 `h5/src/app/api/[[...path]]/route.ts`
  - 捕获所有 `/api/*` 请求
  - 根据环境选择后端地址（生产：`http://134.175.68.92:3001`，开发：`http://localhost:3001`）
  - 在服务器端转发请求到后端，避免混合内容问题
- **文件**：
  - `h5/next.config.ts`：添加 Vercel Rewrites 规则（可能不生效，作为备选方案）
  - `h5/src/app/api/[[...path]]/route.ts`：API 代理路由（主要方案）
- **提交**：`87a5685` - feat: 添加 API 代理路由，将 /api/* 请求转发到后端服务器
- **状态**：已推送到 GitHub，等待 Vercel 自动部署完成
