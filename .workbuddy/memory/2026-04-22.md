# 2026-04-22 工作日志

## 修复：登录页 + 选择小区移动端交互问题

### 问题
1. TopBar「选择小区」按钮点击无反应（移动端）
2. 登录页「获取验证码」按钮点击无反应
3. 登录/注册按钮文案显示「继续使用」，用户困惑

### 根因
- 移动端按钮缺少 `onTouchEnd` 处理，iOS Safari 有 300ms 延迟且偶发 ghost tap 问题
- `paper-surface::after` 伪元素虽已设 `pointer-events: none`，但深层嵌套按钮未继承父级 z-index 提升，加 `relative z-10` 解决
- 弹窗遮罩层用 `<button>` 包裹整个 `inset-0` 区域，在 iOS Safari Webview 有兼容性问题

### 修复内容
**`h5/src/components/TopBar.tsx`**
- 移动端「选择小区」按钮添加 `onTouchEnd`（`e.preventDefault()` + `setOpen(true)`）
- 遮罩层从 `<button>` 改为 `<div>`，添加 `onClick + onTouchEnd` 双重绑定
- 弹窗内容添加 `overscroll-contain` 防滚动穿透
- 关闭按钮 + 弹窗内小区按钮全部添加 `touch-manipulation` + `active:opacity-70`
- 弹窗内小区按钮添加 `onTouchEnd` 双重绑定

**`h5/src/components/CommunityPicker.tsx`**
- 区县选择按钮 + 小区选择按钮添加 `touch-manipulation` + `active:opacity-70`

**`h5/src/app/login/page.tsx`**
- 「继续使用」→ verify 阶段「登录」，profile 阶段「完成注册」
- 加载状态文案：「登录中…」/「提交中…」
- 两个按钮添加 `relative z-10`
- 页面标题改为「登录 / 注册」，副标题和描述文案同步更新
- 新用户提示语：「首次使用，请补全昵称和小区完成注册」
