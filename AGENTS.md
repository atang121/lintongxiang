# AGENTS.md

本项目是“童邻市集”上线版：襄阳东门口周边 5 个相邻小区试点的儿童闲置物品交换 H5。首期不做距离计算和定位排序，默认按小区片区运营。

## 结构

- `h5/`：Next.js 16 + React 19 前端，默认端口 `3000`。
- `server/`：Express + TypeScript API，默认端口 `3001`。
- `server/data/swap.db`：当前 SQLite 业务数据。小范围冷启动可用，正式长期公网运营前要有备份和迁移计划。
- `tools/local-ops-dashboard/`：本机 PM2、构建、健康检查、SQLite 备份可视化面板。

## 常用验证

- 前端契约测试：`cd h5 && node --test test/*.mjs`
- 前端类型检查：`cd h5 && npm exec tsc -- --noEmit`
- 前端构建：`cd h5 && npm run build`
- 后端构建：`cd server && npm run build`
- SQLite 备份：`cd server && npm run backup:sqlite`

注意：当前沙箱环境运行后端 `tsx` 测试/服务时可能出现 IPC pipe 权限问题；能用 `npm run build` 验证编译，运行态测试优先在本机终端或 PM2 面板执行。

运行提醒：当前后端运行的是 `server/dist/index.js` 时，修改 `server/src/**` 后必须先执行 `cd server && npm run build`，再重启 `tonglin-api` 或对应 Node 进程。否则线上仍会跑旧 `dist`，曾导致管理员删除原因和通知没有生效。

## 当前产品规则

- 注册登录使用手机号验证码。孩子年龄段和孩子数量是可选画像，不强制填写。
- 首次注册必须选择小区。内置小区为梧桐湾、清华园、丽江泊林、在水一方、怡和苑；找不到时可手动添加。个人中心修改小区只保留手动填写。
- 账号级《用户服务协议》是当前合规入口。注册/登录需勾选；协议更新后，老用户在发送私信、预约、发布或响应前通过全局弹窗补确认，不强制退出登录。
- 隐私说明使用独立页面 `/privacy`。登录页只在协议勾选行链接《隐私说明》，不要再额外增加底部隐私小字。
- 用户可在个人中心申请注销账号。注销必须是软注销：用户状态置为 `deactivated`，在架和预约中的发布自动下架，历史私信、预约、反馈、违规处理等必要记录保留用于合规、安全、纠纷处理和审计留痕。
- 发布闲置需要至少 1 张图片；发布需求可无图，无图时用 `WantedCover` 生成需求封面。
- 物品详情图使用 `ItemImageCarousel`，多图可滑动，横图/竖图用模糊背景 + `object-contain` 展示。
- 邻里等级按完成交换次数显示：`新邻居`、`有过交换`、`靠谱邻居`、`热心邻居`；不要再展示小数信用评分。
- 交换完成只能由发布者确认。预约者可以取消预约、联系发布者、提醒确认；取消/失败/超时后可重新预约。
- 预约支持候补队列：当前预约优先，候补按时间顺延，轮到时发系统通知。
- 私信只在消息页“私信”里处理，可打开会话并回复；通知页只显示预约/系统类通知，后端不再为私信额外生成 notification。私信内可直接对关联物品预约、重新预约或加入候补。
- 首页有 PWA 添加到桌面提示，使用标准 manifest 和 `/app-icon.svg`；微信内提示用户用右上角/浏览器菜单完成添加。
- 严禁发布宠物或活物，以及协议禁止的食品药品、贴身卫生用品、危险品、侵权仿冒等内容。发布场景识别到此类内容应直接提示禁止发布，不要只提示删敏感词。
- 管理员删除物品必须填写原因；后端应写入 `items.delete_reason/deleted_by/deleted_at` 并给发布者发 `handling` 通知。

## 集成与运维红线

- 不要提交真实 `.env` 或密钥；只维护 `.env.example` / `.env.local.example`。
- 反馈提交支持飞书群机器人、飞书多维表格、邮件兜底，并返回 `providers`、`failed_providers`、`delivery_attempts` 诊断信息。
- 管理员可在后台查看反馈/举报投诉并回复；登录用户在“消息 → 通知”查看反馈处理回复。
- 后台通知运营只保留平台公告、运营提醒、处理通知；交换动态由预约流程自动触发，私信只用角标提示。
- 生产环境短信未配置时不能返回预览验证码；`server/src/services/sms.ts` 已在 `NODE_ENV=production` 下拒绝 preview。真实腾讯云短信需要同时配置 `TENCENT_SECRET_ID`、`TENCENT_SECRET_KEY`、`TENCENT_SMS_SDK_APP_ID`、短信签名和模板 ID。
- 上传只允许安全图片类型和白名单 category，避免写入危险扩展名或越界路径。
- 空生产库不自动灌 demo 数据；演示数据只能用于本地/演示环境。
