> 适用于：腾讯云轻量应用服务器（广州） + Vercel 前端

## 二、服务器初始化（一次性）

用 SSH 连接到你的服务器（腾讯云控制台 → 轻量应用服务器 → 实例 → 登录）：

```bash
# 复制粘贴运行
bash -c "$(curl -fsSL https://raw.githubusercontent.com/你的用户名/lintongxiang/main/docs/服务器初始化脚本.sh)"
```

或者手动运行以下命令：

```bash
# 1. 更新系统
apt update && apt upgrade -y

# 2. 安装 Node.js 18
curl -fsSL https://deb.nodesource.com/setup_18.x | bash -
apt install -y nodejs
node -v   # 确认显示 v18.x.x

# 3. 安装 Git
apt install -y git

# 4. 安装 PM2（进程守护）
npm install -g pm2

# 5. 开放防火墙端口
ufw allow 22/tcp
ufw allow 80/tcp
ufw allow 443/tcp
ufw --force enable
```

## 三、部署后端代码

### 3.1 拉取代码

```bash
mkdir -p /var/www
cd /var/www
git clone https://github.com/你的GitHub用户名/lintongxiang.git lintongxiang
cd lintongxiang/server
```

### 3.2 配置环境变量

```bash
cp .env.production.example .env
nano .env
```

**必须填写的变量（其他可暂时留空）：**

```env
NODE_ENV=production
PORT=3001

# JWT 密钥（随便打一串随机字符串，例如）
JWT_SECRET=随便一串随机字符

# 前端地址（先用 Vercel 给的临时域名，上线后改）
FRONTEND_URL=https://xxx.vercel.app

# SMTP（你的 QQ 邮箱授权码）
SMTP_HOST=smtp.qq.com
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=273576151@qq.com
SMTP_PASS=你的QQ邮箱授权码

# 腾讯云 COS（强烈建议配置）
STORAGE_PROVIDER=cos
COS_SECRET_ID=你的SecretId
COS_SECRET_KEY=你的SecretKey
COS_BUCKET=your-bucket-ap-guangzhou
COS_REGION=ap-guangzhou
```

### 3.3 安装依赖 + 构建

```bash
npm install
npm run build
```

### 3.4 用 PM2 启动

```bash
# 创建日志目录
mkdir -p /var/log/pm2

# 启动（用 ecosystem.config.js）
pm2 start /var/www/lintongxiang/docs/ecosystem.config.js

# 保存启动配置（服务器重启后自动恢复）
pm2 save

# 设置开机自启（根据系统输出提示运行那个命令）
pm2 startup
```

### 3.5 验证服务是否正常

```bash
curl http://localhost:3001/api/health
```

如果输出 `{"status":"ok"...}` 就说明后端跑起来了。

## 四、配置 Nginx（反向代理）

```bash
# 把 Nginx 配置复制过去
sudo cp /var/www/lintongxiang/docs/nginx-lintongxiang.conf /etc/nginx/sites-available/lintongxiang

# 如果用 IP 访问，编辑 server_name 那行（直接用 IP 就跳过这步）
# sudo nano /etc/nginx/sites-available/lintongxiang
# 把 "你的域名" 改成服务器 IP，例如：server_name 1.2.3.4;

# 启用站点
sudo ln -sf /etc/nginx/sites-available/lintongxiang /etc/nginx/sites-enabled/

# 测试配置并重启
sudo nginx -t
sudo systemctl reload nginx
```

现在访问 `http://你的服务器IP/api/health` 看看能不能通。

## 五、部署前端到 Vercel

1. 登录 [vercel.com](https://vercel.com)，用 GitHub 导入仓库
2. Root Directory 选 `h5`
3. Framework 选 `Next.js`
4. Environment Variables 添加：
   - `NEXT_PUBLIC_API_BASE_URL` = `http://你的服务器IP/api`
5. Deploy
6. 部署完成后，把 Vercel 给你的域名（如 `xxx.vercel.app`）**填回后端的** `FRONTEND_URL`，然后在服务器上：

```bash
# 修改后端 .env
nano /var/www/lintongxiang/server/.env
# 把 FRONTEND_URL 改成你的 Vercel 域名

# 重启后端
pm2 restart lintongxiang-api
```

## 六、配置 HTTPS（可选但强烈建议）

需要一个域名（可在阿里云/腾讯云购买，.xyz 域名 10元/年）。

### 6.1 安装 Certbot

```bash
sudo apt install -y certbot python3-certbot-nginx
```

### 6.2 申请 SSL 证书

```bash
sudo certbot --nginx -d your-domain.com
```

按照提示输入邮箱、同意条款即可。Certbot 会自动配置 HTTPS 并设置证书自动续期。

### 6.3 更新 Vercel 的 API 地址

HTTPS 配置好后，在 Vercel 环境变量里把：
```
NEXT_PUBLIC_API_BASE_URL = https://你的域名/api
```
然后重新 Deploy。

## 七、常用运维命令

```bash
# 查看日志
pm2 logs lintongxiang-api

# 重启服务
pm2 restart lintongxiang-api

# 查看状态
pm2 status

# 查看 API 是否正常
curl http://localhost:3001/api/health

# 查看 Nginx 日志
sudo tail -f /var/log/nginx/error.log
sudo tail -f /var/log/nginx/access.log

# 更新代码后重部署
cd /var/www/lintongxiang
git pull
cd server
npm install
npm run build
pm2 restart lintongxiang-api
```

## 八、故障排查

| 问题 | 解决方法 |
|------|---------|
| `curl http://localhost:3001` 无响应 | `pm2 status` 看看进程状态，`pm2 logs` 查错误日志 |
| 502 Bad Gateway | Nginx 没启动：`sudo systemctl restart nginx` |
| SSL 证书过期 | `sudo certbot renew` |
| 数据库丢失 | 检查 `/var/www/lintongxiang/server/data/` 目录是否存在 |
| 图片 404 | 检查 COS 配置是否正确，或者 `/uploads` 目录权限 |
