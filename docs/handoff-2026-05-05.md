# 2026-05-05 阶段交接记录

本记录整理 2026-05-01 至 2026-05-05 期间，围绕“童邻市集”正式试运营前完成的主要审查、修复和体验优化。顺序按本轮对话推进脉络整理，便于后续接手者快速理解为什么这样设计。

## 1. 上线前审查与安全基线

- 项目名称和定位从“邻里童享”统一调整为“童邻市集”，首批面向襄阳东门口周边相邻小区，首期不做距离计算和定位排序。
- 完成上线前代码审查，重点修复短信、上传、预约、公开用户信息、生产 demo 数据、后台路由顺序和用户恢复状态等风险。
- 生产环境短信未完整配置时不再返回 preview 验证码，避免任意手机号被验证码登录。
- 上传链路限制 MIME、扩展名和 category，避免危险扩展名、越界路径和可访问静态文件造成的 XSS/任意写入风险。
- 空生产库不再自动灌入 demo 数据；演示数据只用于本地或演示环境。
- 预约查询、创建、完成、取消、失败均按当前登录身份校验，不再信任客户端传入的 owner_id。
- 公开用户详情脱敏手机号、邮箱、openid、孩子画像等敏感字段。

## 2. 运行、PM2 与本机运维

- 增加本机可视化运维面板 `tools/local-ops-dashboard/`，用于查看 PM2、构建前后端、启动/重启服务、保存 PM2 状态、SQLite 备份和健康检查。
- 明确默认端口：前端 `3000`，后端 `3001`。
- 明确默认 PM2 服务名：前端 `tonglin-h5`，后端 `tonglin-api`。
- 明确 SQLite 数据仍是本机或云服务器项目目录下的 `server/data/swap.db`；如果部署到云服务器，必须在云服务器执行备份，备份才是线上真实数据。
- 新增/完善 SQLite 备份脚本 `server/scripts/backup-sqlite.sh`，备份目录为 `server/backups/sqlite/`。

## 3. 首页、登录注册与小区选择

- 登录页标题文案改为“注册/登录”，注册按钮语义调整为“完成注册并登录”。
- 首次注册保持低摩擦：昵称必填，小区必选，孩子数量和年龄段选填。
- 小区选择采用“5 个内置小区 + 找不到可手动添加”的方案。内置小区为：梧桐湾、清华园、丽江泊林、在水一方、怡和苑。
- 注册页孩子画像改为动态显示：选择 1 个孩子显示 1 组年龄段，选择 2 个显示 2 组，选择 3 个+显示 3 组并提示后续可在个人中心补充。
- 孩子画像仅用于个性化推荐，不公开展示。用户如果选了孩子数量但只填部分年龄段，前端只提交已填写的年龄段，`child_count` 仍记录用户选择的数量。
- 老用户正常登录不展示小区/孩子画像表单，避免登录体验变重。
- 首次登录欢迎弹窗改为更轻、更短的一屏式引导，主动作引导“先逛附近好物”，辅助动作引导发布闲置。

## 4. 发布、图片与物品详情体验

- 发布闲置继续强制至少上传 1 张图片；发布求购需求允许无图。
- 求购需求无图时使用 `WantedCover` 生成视觉封面，不再展示难看的空图占位。
- 物品图片支持多图滑动轮播、缩略图和计数；横图/竖图统一用模糊背景 + `object-contain` 展示，减少比例突兀。
- 发布页支持草稿保存：用户填写内容后跳去阅读协议，返回时不清空已填内容。
- 已发布的闲置/需求支持编辑后重新发布。
- 发布者可主动下架物品；管理员可删除违规物品。
- 价格语义增加“一口价 / 可小刀 / 可商量”等表达，详情文案按出售、以物换物、免费赠送、求购需求区分，不再把所有场景都说成“以物换物”。

## 5. 预约、候补与私信

- 详情页移动端隐藏全局底部导航，由详情页动作栏承接联系、预约、候补、取消、提醒确认、确认交接等核心动作。
- 预约流程改为公平候补：当前预约优先处理，其余按最早留言/预约时间进入候补。
- 预约者可以取消预约或候补；取消、失败、超时后物品恢复可预约，并自动顺延下一位候补。
- 候补用户轮到时会收到系统通知。
- 取消预约后，用户后续仍可重新预约或重新加入候补。
- 私信页和详情页打通：用户在私信聊天中可以直接对关联物品发起预约、重新预约或加入候补，不需要回首页再找物品。
- 私信只在“消息 → 私信”里显示和回复，不再额外生成“你收到一条新私信”的通知，避免通知重复干扰。
- 底部消息角标优化为只展示未读数量，私信和服务通知各归其位。

## 6. 反馈、飞书与管理员回复

- 反馈类型中的“投诉”统一改为“举报投诉”，飞书多维表格字段也按该名称对齐。
- 反馈提交支持三路投递：飞书群机器人、飞书多维表格、邮件兜底；接口返回 `providers`、`failed_providers`、`delivery_attempts` 便于定位失败通道。
- 管理后台增加反馈处理视图，管理员可查看反馈/举报投诉，给已登录用户站内回复；未登录反馈只能根据用户留下的联系方式线下处理。
- 管理员回复反馈后，用户在“消息 → 通知”查看处理回复。
- 反馈回复展示做了去重：后台不再在同一区域重复显示同一条“反馈处理回复”。

## 7. 用户服务协议与合规

- 将分散的《平台定位与免责声明》《闲置发布规则》《求购发布规则》《物品提供承诺》《交易安全须知》整合为一份《用户服务协议》。
- 注册/登录按钮上方显示“我已仔细阅读并同意《用户服务协议》全部条款”，未勾选时按钮动作给 3 秒提示，不堆叠、不长期悬挂。
- 协议详情页改为直接进入单份《用户服务协议》，避免“协议列表 → 协议详情”的多余跳转。
- 阅读协议后支持返回注册/登录或返回继续使用，不再造成登录表单内容丢失。
- 管理后台增加协议管理工具：支持上传 `.docx` / `.txt` / `.md` 或粘贴正文，提取协议文本并发布新版本。
- 后端按当前协议版本判断是否需要用户重新确认。协议更新后，已登录用户在发送私信、预约、发布或响应前会看到全局更新确认弹窗，点击“同意并继续”后无需重新登录。
- 修复过一次协议更新弹窗不关闭的问题：前端 `normalizeUser` 现在优先信任后端返回的 `service_agreement_required`，避免后端已确认但前端用静态版本误判仍需弹窗。

## 8. 禁发内容与敏感词

- 根据《用户服务协议》补充协议风控词库：食品药品、贴身卫生风险、危险品、侵权仿冒、宠物/活物等。
- 明确禁止发布各种宠物或活物信息，包括但不限于鸟、猫、狗、仓鼠等。
- 发布场景中，如果识别到宠物/活物或协议明确禁止内容，直接提示禁止发布，不再只提示“去掉敏感词”。
- 为规避表达增加上下文识别，例如“小伙伴”“不养了”“名字叫”等与活物语境组合时拦截。
- 管理后台支持动态敏感词管理、重新加载词库。
- 管理员删除违规物品时必须填写删除原因，删除原因会写入 `items.delete_reason/deleted_by/deleted_at`，并向发布者发送站内通知。

## 9. 管理后台与通知运营

- 管理后台整体做了操作体验优化，重点分为物品、用户、交换、反馈、敏感词、协议、通知等运营模块。
- 后台通知模块从“发布系统通知”调整为“通知运营 / 发布平台通知”。
- 人工通知类型保留：平台公告、运营提醒、处理通知。
- 支持发送对象：全员、指定小区、指定用户。
- 指定用户发送支持按昵称、电话、小区筛选后勾选多名用户，不需要一个一个精确搜索。
- 交换动态和私信提醒不放在管理员手动群发里：交换动态由预约流程自动触发；私信只用角标提示。
- 通知支持撤回字段 `recalled`，后台可撤回单条或批量通知。

## 10. 视觉、品牌与 PWA

- 平台视觉从早期“童邻”块状 logo 优化为更统一的“童邻市集”品牌页头，并接入方形 logo 作为图标资产。
- 首页首屏文案改为“让孩子的好物继续被需要”，副文案强调绘本、中小学配套读物、玩具、童装等儿童闲置。
- 首页标签使用“东门口周边 · 儿童闲置交换”，小区列表不再重复显示“东门口周边”。
- PWA 添加到桌面提示移动到更合适的位置，主图标使用视觉更强的桌面图标。
- 用户可通过浏览器原生“添加到主屏幕/添加到桌面”能力安装；微信内提示先用右上角菜单或外部浏览器打开。

## 11. 2026-05-05 确认的问题与处理结论

- 2026-05-05 发现“点击协议更新弹窗的同意并继续没有反应”：根因是前端用静态协议版本覆盖后端判断，已修复 `h5/src/lib/normalize.ts`，并通过前端契约测试与生产构建验证。
- 2026-05-05 发现“管理员删除鹦鹉测试信息后用户未收到删除原因”：数据库显示该历史删除记录没有 `delete_reason/deleted_by/deleted_at`，通知表也没有对应通知。原因是删除发生时后端仍在跑旧构建；后续必须在修改后端后执行 `cd server && npm run build` 并重启正在运行的 `dist/index.js` 服务。
- 历史那条鹦鹉测试信息的原始删除原因已经被旧进程丢失，无法从数据库恢复。若需要告知用户，只能由管理员补发一条通用处理通知。

## 12. 2026-05-06 隐私说明与账号注销

- 新增独立隐私说明页 `/privacy`，用于说明手机号、小区、选填孩子画像、浏览/发布/私信/预约/反馈记录等信息的收集目的、使用边界和隐私保护原则。
- 注册/登录协议勾选文案调整为“我已仔细阅读并同意《用户服务协议》全部条款，并知悉《隐私说明》”。《隐私说明》从登录页直接进入 `/privacy?from=login`，阅读后可一键返回注册/登录。
- 按产品确认，不在登录页底部额外展示“我们仅收集必要信息用于邻里安全服务，孩子画像为选填，不公开展示，不出售、不营销。”这句小字，避免登录页信息过载。
- 个人中心菜单增加“隐私说明”入口，方便已登录用户随时查看。
- 个人中心账号设置增加“申请注销账号”。注销采用软注销：`users.status = 'deactivated'`，记录 `status_reason/status_updated_at`，同时自动下架该用户在架和预约中的发布。
- 用户主动注销不会物理删除历史私信、预约、反馈、违规处理等必要记录，保留目的仅限法律法规、安全风控、纠纷处理和审计留痕。
