## 一、已修复的 Bug（3 个）

### 1. 🔴 admin.ts 缺少 `getOne` 导入（高）
- **文件**：`server/src/routes/admin.ts` L460
- **问题**：`PUT /api/admin/exchanges/:id/status` 路由使用了 `getOne()` 但 import 语句中未导入
- **影响**：管理员修改交换状态时会抛 `ReferenceError: getOne is not defined`，整个交换管理功能无法使用
- **修复**：在 import 语句中添加 `getOne`

### 2. 🔴 品牌名残留"童邻市集"（高）
- **文件**：`server/src/index.ts`、`server/src/services/mail.ts`（4处）、`server/src/services/feishuWebhook.ts`、`server/src/demo/seeds.ts`、`server/.env`
- **问题**：旧品牌名未完全替换为"童邻市集"，邮件模板、飞书通知、SMTP 发件人名称仍显示旧名
- **影响**：用户收到的验证码邮件标题显示"童邻市集验证码"，飞书通知显示"童邻市集收到新反馈"
- **修复**：全部替换为"童邻市集"

### 3. 🟡 rateLimit.ts `ipKeyGenerator` 参数类型错误（中）
- **文件**：`server/src/middleware/rateLimit.ts` L64、L81
- **问题**：`ipKeyGenerator` 签名为 `(ip: string)` 但代码传入了整个 `Request` 对象
- **影响**：TypeScript 编译错误，运行时可能生成错误的限流 key 导致限流失效
- **修复**：改为 `ipKeyGenerator(req.ip || '')`

## 二、待配置项（上线前必须完成）

### 1. JWT_SECRET 必须更换为强随机密钥
- **当前**：`xiangyang_kids_swap_dev_secret_2024_change_in_production`（开发弱密钥）
- **要求**：至少 32 字符随机字符串
- **生成**：`openssl rand -hex 32`
- **不配置后果**：生产环境拒绝启动（session.ts L12-13）

### 2. ADMIN_TOKEN 需设置
- **当前**：`.env` 中未配置
- **用途**：管理后台 `X-Admin-Token` 认证（备用方式）
- **建议**：`openssl rand -hex 16`

### 3. NODE_ENV 改为 production
- **当前**：`NODE_ENV=development`
- **影响**：
  - 开发模式下 CORS 允许所有来源
  - 快捷登录 `/api/auth/login` 仅在 development 可用（生产已禁用 ✅）
  - Cookie secure 标志在 production 才启用

### 4. FRONTEND_URL 改为生产域名
- **当前**：`http://localhost:3000`
- **要求**：改为实际部署域名

### 5. SERVER_PUBLIC_URL 改为实际地址
- **当前**：`https://your-domain.com`
- **用途**：图片 URL 拼接、微信分享签名

### 6. 短信服务配置（如需真实短信登录）
- 当前为 preview 模式（验证码直接返回给前端）
- 需要配置：`SMS_ENABLED=true` + 腾讯云 SMS 四项参数
- ⚠️ **注意**：`sms.ts` 中凭证映射有误 — `TENCENT_SMS_APP_ID` 被同时用作 API SecretId 和 SmsSdkAppId，腾讯云这两个是完全不同的值。建议增加 `TENCENT_SECRET_ID` 和 `TENCENT_SECRET_KEY` 两个独立配置项

### 7. 图片存储（如需 OSS）
- 当前默认本地存储 `server/data/uploads/`
- 如需腾讯云 COS：配置 `COS_SECRET_ID`、`COS_SECRET_KEY`、`COS_BUCKET`、`COS_REGION`
